# Ontahí - System Architecture

This is the first Ontahí book, although it does not begin with Ontahí.

It begins with systems.

The central thesis is simple:

> Software systems are living systems.

Architecture is not the pursuit of perfection. Architecture is the discipline of managing the cost of change.

A good architecture is not one that perfectly models today's reality. It is one capable of absorbing tomorrow's ideas without collapsing.

## Writing Posture

This is not marketing.

This is not academic writing.

This is not product documentation.

The tone should be conceptual, reflective, simple, technical when needed, opinionated but honest. Every chapter should teach an idea, not a feature.
